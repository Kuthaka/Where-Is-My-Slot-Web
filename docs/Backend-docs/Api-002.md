# Where Is My Slot - Backend API Documentation v2

**Base URL**: `http://localhost:5000` (or `https://api.whereslot.shamveelp.xyz` in production)

## Authentication

Most protected endpoints require a Bearer token in the `Authorization` header.

**Header Format:**
```
Authorization: Bearer <your_jwt_access_token>
```

---

## 1. System & Health

### 1.1 Health Check
Check if the API is running.
- **URL**: `/api/v1/health`
- **Method**: `GET`
- **Auth Required**: No
- **Response Format**:
  ```json
  {
    "status": "string (e.g. 'ok where slot')",
    "timestamp": "string (ISO Date)",
    "version": "string"
  }
  ```

### 1.2 Seed Database
Seed the database with initial test data (User, Category, Business).
- **URL**: `/api/v1/seed`
- **Method**: `POST`
- **Auth Required**: No
- **Response Format**: Returns the seeded `user`, `category`, and `business` objects.

## 2. User Authentication (`/api/v1/auth`)

### 2.1 Send OTP
Sends a 6-digit OTP to the user's email for verification.
- **URL**: `/api/v1/auth/send-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "string (required, valid email)"
  }
  ```
- **Response Format**: 
  ```json
  {
    "success": "boolean",
    "data": {
      "message": "string"
    }
  }
  ```

### 2.2 Verify OTP
Verifies the OTP sent to the user's email.
- **URL**: `/api/v1/auth/verify-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "string (required)",
    "otp": "string (required, exactly 6 characters)"
  }
  ```
- **Response Format**: 
  ```json
  {
    "success": "boolean",
    "data": {
      "message": "string (contains JWT token on success)"
    }
  }
  ```

### 2.3 User Login
Login with email and password.
- **URL**: `/api/v1/auth/login`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "string (required, valid email)",
    "password": "string (required)"
  }
  ```
- **Response Format**:
  ```json
  {
    "success": "boolean",
    "data": {
      "accessToken": "string (JWT)",
      "user": {
        "id": "string",
        "name": "string",
        "username": "string (optional)",
        "email": "string",
        "isPasswordSet": "boolean",
        "role": "string (USER or BUSINESS or SUPER_ADMIN)",
        "isActive": "boolean",
        "location": {
          "address": "string",
          "latitude": "number",
          "longitude": "number"
        },
        "createdAt": "string (ISO Date)",
        "updatedAt": "string (ISO Date)"
      }
    }
  }
  ```

### 2.4 User Register
Register a new user account. Requires a verified OTP in the payload.
- **URL**: `/api/v1/auth/register`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "firstName": "string (required)",
    "lastName": "string (required)",
    "username": "string (required, min 3 chars, alphanumeric)",
    "email": "string (required, valid email)",
    "password": "string (required, min 6 chars)",
    "otp": "string (required, exactly 6 characters)"
  }
  ```
- **Response Format**: Returns the same structure as `Login` (`accessToken` and `user`).

### 2.5 Check Availability
Check if a username or email is already taken.
- **URL**: `/api/v1/auth/check-availability`
- **Method**: `GET`
- **Query Params**: `?username=string` OR `?email=string`
- **Auth Required**: No
- **Response Format**:
  ```json
  {
    "success": "boolean",
    "data": {
      "available": "boolean"
    }
  }
  ```

### 2.6 Set Password
Change or set the user's password.
- **URL**: `/api/v1/auth/set-password`
- **Method**: `POST`
- **Auth Required**: Yes (Bearer Token)
- **Request Body**:
  ```json
  {
    "newPassword": "string (required, min 6 chars)",
    "oldPassword": "string (optional, required if password was already set)" 
  }
  ```
- **Response Format**: 
  ```json
  {
    "success": "boolean",
    "data": {
      "message": "string"
    }
  }
  ```

### 2.7 Get Current User (Me)
Get the profile of the currently authenticated user.
- **URL**: `/api/v1/auth/me`
- **Method**: `GET`
- **Auth Required**: Yes (Bearer Token)
- **Response Format**: Returns the `user` object (see Login response).
