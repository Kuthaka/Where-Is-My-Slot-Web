# Where Is My Slot - Backend API Documentation

**Base URL**: `https://api.whereslot.shamveelp.xyz`

## Authentication

Most protected endpoints require a Bearer token in the `Authorization` header.

**Header Format:**
```
Authorization: Bearer <your_jwt_access_token>
```

---

## 1. System & Health

### 1.1 Health Check
Check if the API is running.
- **URL**: `/api/v1/health`
- **Method**: `GET`
- **Auth Required**: No
- **Success Response**:
  ```json
  {
    "status": "ok where slot",
    "timestamp": "2026-07-15T12:00:00.000Z",
    "version": "2.0.0"
  }
  ```

### 1.2 Seed Database
Seed the database with initial test data (User, Category, Business).
- **URL**: `/api/v1/seed`
- **Method**: `POST`
- **Auth Required**: No
- **Success Response**: Returns the seeded `user`, `category`, and `business` objects.

## 2. User Authentication (`/api/v1/auth`)

### 2.1 Send OTP
Sends a 6-digit OTP to the user's email for verification.
- **URL**: `/api/v1/auth/send-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "user@example.com"
  }
  ```
- **Success Response**: `200 OK` with success message.

### 2.2 Verify OTP
Verifies the OTP sent to the user's email. Used for passwordless login or verification.
- **URL**: `/api/v1/auth/verify-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "user@example.com",
    "otp": "123456"
  }
  ```
- **Success Response**: Returns a JWT access token on successful verification.

### 2.3 User Login
Login with email and password.
- **URL**: `/api/v1/auth/login`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "user@example.com",
    "password": "password123"
  }
  ```
- **Success Response**:
  ```json
  {
    "success": true,
    "data": {
      "accessToken": "jwt_token_here",
      "user": { ...user_details }
    }
  }
  ```

### 2.4 User Register
Register a new user account. Requires a verified OTP in the payload.
- **URL**: `/api/v1/auth/register`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "firstName": "John",
    "lastName": "Doe",
    "username": "johndoe",
    "email": "user@example.com",
    "password": "password123",
    "otp": "123456"
  }
  ```
- **Success Response**: Returns the created `user` object and `accessToken`.

### 2.5 Check Availability
Check if a username or email is already taken.
- **URL**: `/api/v1/auth/check-availability`
- **Method**: `GET`
- **Query Params**: `?username=johndoe` OR `?email=user@example.com`
- **Auth Required**: No

### 2.6 Get Current User (Me)
Get the profile of the currently authenticated user.
- **URL**: `/api/v1/auth/me`
- **Method**: `GET`
- **Auth Required**: Yes (Bearer Token)
- **Success Response**: Returns the `user` object.

### 2.7 Set Password
Change or set the user's password.
- **URL**: `/api/v1/auth/set-password`
- **Method**: `POST`
- **Auth Required**: Yes (Bearer Token)
- **Request Body**:
  ```json
  {
    "newPassword": "newpassword123",
    "oldPassword": "oldpassword123" // Optional if setting for the first time
  }
  ```

## 3. Business Authentication (`/api/v1/business-auth`)

### 3.1 Send Business OTP
Send an OTP to a business email for registration/onboarding.
- **URL**: `/api/v1/business-auth/send-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "business@example.com"
  }
  ```

### 3.2 Verify Business OTP
Verify the OTP for business registration. Returns a short-lived session token used for onboarding.
- **URL**: `/api/v1/business-auth/verify-otp`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "business@example.com",
    "otp": "123456"
  }
  ```
- **Success Response**: Returns `{ "verified": true, "verifiedToken": "jwt_token" }`.

### 3.3 Business Login
Login as a merchant/business owner.
- **URL**: `/api/v1/business-auth/login`
- **Method**: `POST`
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "email": "business@example.com",
    "password": "password123"
  }
  ```

### 3.4 Business Set Password
Change or set the merchant password.
- **URL**: `/api/v1/business-auth/set-password`
- **Method**: `POST`
- **Auth Required**: Yes (Bearer Token)

## 4. Businesses (`/api/v1/businesses`)

### 4.1 Explore Businesses
Get a list of approved businesses.
- **URL**: `/api/v1/businesses/explore`
- **Method**: `GET`
- **Auth Required**: No

### 4.2 Get My Business
Get the business profile owned by the authenticated merchant.
- **URL**: `/api/v1/businesses/me`
- **Method**: `GET`
- **Auth Required**: Yes (Bearer Token, Role: `BUSINESS` or `SUPER_ADMIN`)

### 4.3 Update My Business
Update the business profile of the authenticated merchant.
- **URL**: `/api/v1/businesses/me`
- **Method**: `PATCH`
- **Auth Required**: Yes (Bearer Token, Role: `BUSINESS` or `SUPER_ADMIN`)

### 4.4 Onboard Business
Submit business details to complete the registration/onboarding process.
- **URL**: `/api/v1/businesses/onboard`
- **Method**: `POST`
- **Auth Required**: No (but typically requires the `verifiedToken` from OTP verification)

### 4.5 Upload Image
Upload an image to Cloudinary (used for business profiles, posts, etc.).
- **URL**: `/api/v1/businesses/upload-image`
- **Method**: `POST`
- **Auth Required**: No
- **Headers**: `Content-Type: multipart/form-data`
- **Request Body**: Form-data with key `file` (the image file).
- **Success Response**: Returns the secure URL of the uploaded image.

## 5. Admin (`/api/v1/admin`)

*Note: All Admin routes require a Bearer Token with the `SUPER_ADMIN` role.*

### 5.1 Get All Businesses
Retrieve all businesses (pending, approved, rejected).
- **URL**: `/api/v1/admin/businesses`
- **Method**: `GET`
- **Auth Required**: Yes (`SUPER_ADMIN`)

### 5.2 Add Business (Admin)
Create a new business directly as an admin.
- **URL**: `/api/v1/admin/businesses/add`
- **Method**: `POST`
- **Auth Required**: Yes (`SUPER_ADMIN`)

### 5.3 Approve Business
Approve a pending business registration.
- **URL**: `/api/v1/admin/businesses/:id/approve`
- **Method**: `POST`
- **Auth Required**: Yes (`SUPER_ADMIN`)

### 5.4 Reject Business
Reject a pending business registration.
- **URL**: `/api/v1/admin/businesses/:id/reject`
- **Method**: `POST`
- **Auth Required**: Yes (`SUPER_ADMIN`)
